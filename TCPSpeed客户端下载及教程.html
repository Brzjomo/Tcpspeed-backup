TCPSpeed客户端Java版,支持Linux,Windows,OS X,支持命令行运行

下载地址: tcpspeed_client3.2.zip

系统需安装java运行环境,Linux还需安装libpcap,Windows下为winpcap.

Ubuntu,Debian安装libpcap: apt-get -y install libpcap-dev

Centos安装libpcap: yum -y install libpcap

安装:下载解压.

运行: 假设tcpspeed_client.jar所在路径为/tsclient ,先切换到该路径cd /tsclient ,

然后执行 sudo java -jar tcpspeed_client.jar ,前面加sudo,因为必须以root权限运行.

Windows直接执行start_client.bat.

命令行运行客户端教程,针对无图形界面系统

1,首先运行图形界面版客户端,设置并登录成功,然后打开tcpspeed客户端所在目录,可见配置文件client_config.json,port_map.json.

2.将配置文件复制到需要命令行运行的机器上,和tcpspeed_client.jar同一目录.

3.切换到tcpspeed_client.jar所在目录,执行 sudo java -jar tcpspeed_client.jar -b ,如果系统为windows ,执行start_client_cmd.bat.

加速教程-通过内置代理加速

支持加速游戏,不依赖第三方软件.

将需要加速的软件代理设置为socks5 127.0.0.1:1083 .

如果不会设置,可下载下面的代理工具.

Proxifier 默认所有浏览器走代理

SocksCap64 将要代理的软件拖入Sockscap64窗口完成添加,并从SocksCap64启动
由于硬盘坏掉所有数据丢失,SSTAP和SOCKSCAP64将永久停止开发--11/18/2017

注意: 客户端默认socks5端口为1083,如果修改过该端口号,其它地方也需相应修改.

加速教程-通过映射加速

假设服务器地址为5.5.5.5,需要加速的端口为888(通常ssh端口为22,远程桌面为3389,mysql为3306),将其映射到本地端口2000 (端口号随意)

如果为其他地址和端口,请作相应修改.

1.首先填写tcpspeed登录信息

2.点击添加,增加映射,本地端口为2000,目标地址为服务器地址,目标端口为需要加速的端口号.

3.运行你的客户端,新建连接,并设置连接信息.

只需将服务器地址改为127.0.0.1,端口改为本地端口,其他配置信息和直连一样,不用做改动.
